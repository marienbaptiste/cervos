## Network
- **Tailscale mesh**: all traffic encrypted end-to-end via WireGuard
- No ports exposed to the public internet
- Phone ↔ Mac Studio communicate exclusively over Tailscale

## mTLS
- CA + server cert + client cert generated by `scripts/generate-certs.sh`
- Certs stored in `server/nginx/certs/` (gitignored — never committed)
- Only the registered Flutter app client cert is accepted by nginx

## CI/CD
- GitHub Actions workflows in `.github/workflows/`
- `ci.yml`: config validation, docker-compose validation, SQLite migration checks, Flutter analyze, firmware source checks
- `design-lint.yml`: validates design system tokens and components on changes to `design-system/`

## Scripts
```
scripts/
├── setup.sh            # Full setup orchestrator
├── setup-native.sh     # mlx-lm, whisper, Ollama (delegates to server/native-setup.sh)
├── setup-docker.sh     # Docker Compose services
├── generate-certs.sh   # mTLS CA + server + client certificates
├── tailscale-setup.sh  # Tailscale mesh enrollment
└── test-e2e.sh         # End-to-end smoke test (all services)
```

## Security (PUBLIC REPO)
- `config.yaml` is gitignored — contains API keys and Tailscale auth
- Only `config.example.yaml` is committed (empty placeholder values)
- Certificates are generated locally, never committed
- `server/nginx/certs/` contents are gitignored
- `data/` directory is gitignored (SQLite, audit logs, Chroma data)
- Never hardcode IPs, passwords, tokens, or keys in any tracked file
- If you create a new file that might contain secrets, add it to `.gitignore`
